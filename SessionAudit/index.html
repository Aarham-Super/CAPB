CAPB SYSTEM AUDIT

SessionAudit

SessionAudit provides full transparency into CAPB’s architecture, security flow, and verification logic.

Purpose

SessionAudit exists to document how CAPB processes authentication data and verification signals.

System Flow

  • User interacts with HoverShield
  • Tokens are generated (Turnstile + reCAPTCHA)
  • EdgeGuard validates both tokens
  • Final result is returned

Privacy Model

CAPB does not store user data. All verification requests are processed in real time and discarded immediately.
